# SubInsights — Technical Specification (Reverse-Engineered from Build)

> This spec was reverse-engineered from the final working dashboard. All decisions, gotchas, and implementation details are documented here — so a developer can build this in one shot without the 5-hour iteration cycle.

---

## 1. Concept

**What it does:** Single-page dashboard that connects to RevenueCat Charts API and generates a weekly growth memo.

**Product thesis:** "Memo, not dashboard." Founders need a verdict + findings + actions, not another tab to open.

**Target user:** Indie subscription app founders who check RevenueCat every Monday.

**Live demo:** https://subinsights-pi.vercel.app (Dark Noise — real iOS app, real RC data)

---

## 2. Stack

- **Framework:** Next.js 14, App Router, TypeScript strict
- **Charts:** Recharts (`AreaChart`, `BarChart`, `Funnel策`)
- **Memo generation:** GPT-4o-mini (falls back to rule-based if no API key)
- **Styling:** Plain CSS inline styles — NO Tailwind, NO CSS modules
- **Hosting:** Vercel (free tier)
- **Data:** RevenueCat Charts API v2

## 11. Data Architecture

### ISR Strategy

```
page.tsx (server component):
  - fetchDashboardData("all")
  - fetchDashboardData("1y")
  - fetchDashboardData("3m")
  - fetchDashboardData("1m")
  - generateMemo(metrics)
  - revalidate = 3600 (1 hour)
  
DashboardClient (client component):
  - receives all 4 periods as props
  - useState for selected period
  - switching = instant, no API call
```

### API Endpoints Called (server-side only, NEVER in browser)

```
GET /v2/projects/{id}/metrics/overview
GET /v2/projects/{id}/charts/mrr?resolution={day|week|month}&start_time=...&end_time=...
GET /v2/projects/{id}/charts/revenue?resolution=day&start_time=...&end_time=...
GET /v2/projects/{id}/charts/churn?resolution=day&start_time=...&end_time=...
GET /v2/projects/{id}/charts/trial_conversion_rate?resolution=day&start_time=...&end_time=...
```

### Measure Index Mappings

⚠️ CRITICAL GOTCHA: `trial_conversion_rate` chart returns 5 measures:
```
measure[0] = Trial Starts
measure[1] = Conversions
measure[2] = Expirations ← NOT conversions!
measure[3] = Pending
measure[4] = Conversion Rate %
```
**Use measure[0] for Trial Starts, measure[1] for Conversions. NEVER use measure[2] for conversions.**

### MRR Resolution by Period

```
1M  → day
3M  → week
1Y  → month
ALL → month
```

### Incomplete Data Flag

Chart data has `incomplete: boolean` per value. Filter out incomplete values from the end of the series (keep historical, discard current partial period).

### Rate Limit Handling

RC Charts API: 15 req/min. Solution: pre-fetch all 4 periods at build time, embed in static HTML. End users get zero runtime API calls. ISR handles refresh once/hour.

## 12. Fallback Architecture

When RC API is unavailable (429, network error, or no API key):

1. Set `USE_MOCK_DATA=true` in environment or catch block
2. Read from local JSON files in `/data` directory:
   - `overview.json` — 4 metric values
   - `mrr_1m.json`, `mrr_3m.json`, `mrr_1y.json`, `mrr_all.json` — each period's MRR series
   - `revenue_day.json`, `churn_day.json`, `trial_conv.json` — chart data
3. When no OpenAI key: use `generateFallbackMemo()` — rule-based memo with verdict + bullets + actions

### Mock Data Structure

```typescript

{ "metrics": [{ "id": "mrr", "name": "MRR", "value": 4554, "unit": "usd" }, ...] }

// mrr_1m.json (day resolution)

  "values": [{ "cohort": 0, "measure": 0, "value": 4500, "incomplete": false }, ...],
  "measures": [{ "display_name": "MRR", "unit": "usd", "chartable": true }],
  "summary": {}
}
```

## 13. Memo Generation Prompt

```
You are a subscription analytics advisor for indie app founders.
Write a weekly growth memo for {appName}.

Context: {isStable ? "This week is stable..." : "This week shows..."}

Data: MRR ${mrr} ({wow}% WoW), Active subs {subs}, Revenue ${revenue}, Churn {churn}% ({churnWow}% WoW), Trial conv {trialConv}% ({trialWow}% WoW).

Format: 1) verdict sentence, 2) 3 bullet findings, 3) 2 actions. Max 150 words.

Rules:
- If MRR stable (±1%) and churn < 5%, say "stable week", not "positive momentum"
- Never make up numbers
- Actions must be specific: "Test trial screen copy" not "Improve onboarding"
```

Fallback memo (rule-based):
- Stable = MRR WoW < 1% AND churn < 5%
- Verdict: 🟢/🟡/🔴 based on MRR direction
- 3 bullets from data
- 2 specific actions based on context

## 14. Security Model

- `RC_API_KEY` and `RC_PROJECT_ID` are server-side only (Next.js server component)
- `OPENAI_API_KEY` is server-side only
- Zero `NEXT_PUBLIC_` prefixed secrets
- `.env.example` in public repo (no real keys)
- `lib/revenuecat.ts` never imported in client components
- `.gitignore` excludes `.env`
